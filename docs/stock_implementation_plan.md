# 개발 계획서 - 프리미엄 주식 트래커 및 시뮬레이터

우리는 가상 주가 시뮬레이션 기능과 모의 투자 거래 엔진이 포함된 프리미엄 주식 트래커 웹 어플리케이션을 구축할 것입니다. HTML, CSS, JavaScript를 기반으로 하며 차트 렌더링을 위해 Chart.js 라이브러리를 활용합니다.

---

## 사용자 검토 요구사항 (User Review Required)

> [!IMPORTANT]
> **기존 파일 덮어쓰기**: 기존에 작성된 투두리스트 앱 대신 해당 작업 폴더에 주식 앱을 새로 구축합니다. 따라서 루트 경로의 `index.html`, `index.css`, `app.js` 파일이 주식 앱 코드로 교체됩니다.

#### [NEW] [app.js](file:///c:/Users/user/Desktop/new/app.js)
- **주가 시뮬레이션 엔진**: 3~5초마다 무작위 Walk 모델을 이용하여 각 종목의 주가를 변동시키고 실시간으로 등락률을 갱신하는 타이머 구현.
- **Chart.js 연동**: 각 주식의 역사적 가상 주가 데이터를 보관하고 기간 필터(1D, 1W, 1M, 1Y)에 따라 반응형 라인 차트 렌더링.
- **모의 투자 거래 로직**:
  - 초기 자산 $10,000 기준 매수/매도 주문 및 평균 단가 산출 로직.
  - 보유 현금 및 주식 평가 가치를 합산하여 실시간 순자산가치(NAV) 및 손익률(P&L)을 갱신.
  - 원장 관리 (거래 체결 내역 배열 추가).
- **관심 종목 및 뉴스**: 관심 종목 지정 토글 및 뉴스 카드 동적 생성.
- **영속성**: `localStorage`를 통해 자산 현황, 거래 내역, 관심 종목을 브라우저에 저장하고 불러옴.

## 검증 계획 (Verification Plan)

### 수동 검증 사항
- **실시간 주가 변동**: 백그라운드 엔진에 의해 3~5초 주기로 주가가 역동적으로 바뀌고 변동에 따라 녹색/적색 플래시 애니메이션이 발생하는지 확인.
- **차트 상호작용**: 종목을 선택했을 때 차트가 갱신되는지, 기간 필터를 누를 때 차트가 다시 그려지는지 확인.
- **모의 투자 거래 테스트**:
  - $10,000 한도 내에서 주식을 정상적으로 매수할 수 있는지 확인.
  - 보유 수량 한도 내에서 매도 처리가 정상적으로 완료되고, 평가 이익 및 손익률이 실시간 반영되는지 확인.
  - 거래 원장에 기록이 남는지 확인.
- **로컬 스토리지 연동**: 페이지를 새로고침한 뒤에도 잔고, 보유 주식, 거래 내역이 온전히 보관되어 있는지 검증.
